# CIDOC CRM

**Concern:** Semantic integration and exchange of cultural heritage information across museums, libraries, and archives.
**Issuing body:** ICOM Documentation / CIDOC CRM SIG (formal ISO standard since 2006).
**Scope:** Extensible ontology — 81 classes, 160 properties — for modelling collection records, provenance, events, agents, and places.

## When to cite

Cite when a pipeline designs, describes, or exchanges cultural heritage records: museum catalogues, archival finding aids, provenance chains, inter-institution data exchange. Map collection objects to specific CIDOC entity types (e.g., `E24 Physical Human-Made Thing`, `P108i was produced by`) — cite the class/property, not just "CIDOC".

## Notes

RDF/XML and OWL implementations available. Extension modules for archaeology, geoinformatics, bibliographic records. Complements Dublin Core (which handles resource-level metadata) at the deeper semantic layer.
