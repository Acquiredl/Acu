# ISO/IEC 25010 — Software Quality Model

**Concern:** Taxonomy of software quality characteristics for design, evaluation, and comparison.
**Issuing body:** ISO / IEC, part of the SQuaRE (Systems and software Quality Requirements and Evaluation) family.
**Scope:** Nine top-level characteristics — Functional Suitability, Performance Efficiency, Compatibility, Interaction Capability, Reliability, Security, Maintainability, Portability, Safety — each with sub-characteristics.

## When to cite

Cite when a pipeline designs, evaluates, or compares software quality. Pick a specific characteristic and sub-characteristic (e.g., "25010 Maintainability — Modularity"), not just "25010".

## Notes

2023 edition added Safety and reorganised Usability into Interaction Capability. Complements testing-method standards; 25010 defines *what* to measure, not *how*.
