# NIST SSDF (SP 800-218)

**Concern:** Secure software development lifecycle practices — organisational readiness, software protection, well-formed production, defect review.
**Issuing body:** NIST.
**Scope:** 19 practices and 42 tasks in four groups — PO (Prepare Organisation), PS (Protect Software), PW (Produce Well-formed software), RV (Review defects).

## When to cite

Cite when a pipeline designs a secure-by-default SDLC for a software product. Cite a specific practice (e.g., "SSDF PW.7 — Review human-readable code for vulnerabilities").

## Notes

Complements (does not replace) NIST CSF — SSDF is SDLC-layer, CSF is organisation-wide governance. Mandated for US federal procurement (OMB M-22-18).
